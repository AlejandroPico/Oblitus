Sobre la web

Oblitus est scientia nace como una biblioteca personal de red divulgativa: artículos largos, investigaciones, ensayos, documentos convertidos en publicaciones web y piezas interactivas.

Alejandro Picó

Soy Alejandro Picó, de Barcelona. Desde siempre he sentido una atracción profunda por la tecnología, los avances científicos, la política y todas aquellas disciplinas que ensanchan los límites de lo que entendemos como conocimiento. Oblitus nace como un espacio personal desde el que explorar, escribir y reflexionar sobre los temas que me interesan: la inteligencia artificial, la programación, las matemáticas, la física, la química, el arte, la historia militar, la geopolítica y, en general, cualquier ámbito que permita mirar la realidad con más profundidad.

Este proyecto surge también de una curiosidad muy concreta: la fascinación por el espacio, la astrofísica, los cohetes, la exploración científica, los grandes desafíos técnicos y las preguntas difíciles. Oblitus est scientia quiere recoger esa mezcla de asombro, estudio, análisis y frikismo bien entendido: una forma de mirar el mundo con entusiasmo, pero también con rigor, estructura y voluntad constante de aprender.

La intención no es publicar contenido rápido ni superficial, sino construir poco a poco un archivo propio de artículos, ensayos, proyectos y piezas interactivas donde la ciencia, la tecnología y el pensamiento crítico puedan convivir con una estética cuidada y una línea editorial reconocible.

Línea editorial

La página está pensada para publicar contenido con tono serio, estructura documental y vocación divulgativa. No busca funcionar como una red social ni como un blog rápido de actualidad, sino como un archivo ordenado de textos extensos sobre ciencia, tecnología, pensamiento crítico, historia, geopolítica del conocimiento e inteligencia artificial.

La prioridad visual es que el lector entre, localice la biblioteca y pueda abrir un artículo con el menor número posible de pasos. Las páginas explicativas quedan separadas para no cargar la portada con bloques institucionales innecesarios.

Temáticas base

Las temáticas sirven para ordenar la biblioteca y mantener una línea coherente entre publicaciones. No son una limitación cerrada, sino un mapa inicial de intereses que irá creciendo con el propio proyecto.

  • Inteligencia artificial.
    Modelos, agentes, automatización, ética, impacto social y cultura técnica.
  • Física.
    Cosmología, partículas, mecánica cuántica, astrofísica y fundamentos de la materia.
  • Matemáticas.
    Conceptos, historia, visualización, aplicaciones y herramientas de aprendizaje.
  • Tecnología.
    Software, programación, sistemas, redes, plataformas y cultura digital.
  • Historia de la ciencia.
    Ideas, descubrimientos, controversias, biografías y evolución del conocimiento.
  • Geopolítica del conocimiento.
    Ciencia, poder, industria, defensa, economía y soberanía tecnológica.
  • Cultura crítica.
    Ensayos de interpretación, pensamiento contemporáneo y análisis de narrativas públicas.
  • Microinterfaces.
    Simulaciones JavaScript, visualizaciones interactivas y piezas experimentales.

Criterio de etiquetas

Las etiquetas deben servir para localizar artículos, no para decorar. La recomendación inicial es usar pocas etiquetas por publicación, entre tres y seis, combinando una materia principal, una submateria y un tipo de enfoque.

Materia

IA, Física, Matemáticas, Tecnología, Historia, Geopolítica, Cultura crítica.

Submateria

Cosmología, LLM, algoritmos, energía, partículas, redes, programación, automatización.

Formato

Ensayo, investigación, guía, visualización, simulación, reseña, apunte técnico.

Sistema de publicación

Los documentos fuente se guardan en la carpeta articulos/. El proyecto mantiene una estructura estática para GitHub Pages: los documentos se transforman en HTML y el índice generado alimenta la portada.

  1. Se escribe o prepara un documento en Word, PDF, Markdown o HTML.
  2. El archivo se guarda dentro de articulos/.
  3. El generador construye el índice de publicaciones y los HTML internos.
  4. La portada muestra los artículos como mosaico ordenado por fecha.
  5. El lector interno abre el contenido como una publicación propia de la web.

Formatos previstos

Word / DOCX

Formato recomendado para artículos largos. Funciona mejor usando estilos reales: títulos, subtítulos, listas y tablas.

PDF

Útil para conservar documentos finales, aunque no siempre respeta la estructura al convertirlo a artículo web.

HTML interactivo

Formato indicado para publicaciones con animaciones, simulaciones o pequeñas microinterfaces JavaScript.

Markdown

Opción ligera para textos limpios, notas editoriales y documentos que no requieran maquetación compleja.

Prompt Maestro Publicación

Este recurso define las instrucciones maestras para convertir documentos de Word, PDF, Markdown, HTML o EPUB en artículos compatibles con Oblitus est scientia. Está pensado para pegarse en una conversación de inteligencia artificial y obtener una carpeta de publicación lista para subir a articulos/.

Conversión editorial asistida por IA

El prompt exige HTML semántico, metadatos editoriales, ausencia de estilos globales, estructura por encabezados, limpieza de párrafos, referencias ordenadas y compatibilidad con futuras funciones de lectura, impresión, exportación y microinterfaces.
